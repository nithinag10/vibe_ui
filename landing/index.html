How it works

FAQ

Things people ask.

Is my data safe?

Vibe stores your API key and your page customizations locally in chrome.storage.local. There are no Vibe servers. When you submit a prompt, your prompt and a DOM snapshot (tag names, classes, visible text — never form values or passwords) are sent to the AI provider you configured. That's it.

Do I need an API key?

Yes. You bring your own — today that's a key from console.anthropic.com; more providers are on the way. The key stays on your device and is only used to talk to your agent directly.

What websites does it work on?

Any site you can visit. Vibe injects a button on every page and can restyle/restructure anything the page renders — including SPAs like YouTube, Twitter, Reddit, Gmail.

Does it break websites?

It only adds CSS and scoped JS. You can undo any change, reset the page entirely, or jump back to a previous version. Nothing is destructive.

How much does it cost?

Vibe itself is free and open-source. You only pay your AI provider for what you send — typically fractions of a cent per vibe. Long sessions are auto-compacted to keep token use low.

Is it Chrome-only?

For now, yes — it's a Chrome MV3 extension. It should work on any Chromium browser that supports MV3 extensions (Arc, Brave, Edge). Firefox support isn't there yet.

Can I contribute?

Please do. CONTRIBUTING.md walks through the codebase. Adding a new tool is a single file.
